#### 1. Token (令牌/词元)
*   **我理解是**：大模型不认识人类的文字（汉字或英文字母），它只认识数字。Token 就是文字和数字之间的“桥梁”。你可以把它当成一个==“切碎的词根”==。
*   *举个例子*：单词 "hamburger" 可能会被切成 "ham" 和 "burger" 两个 Token。一般情况下，==1个汉字大约是 1-2 个 Token==。
*   **为什么重要**：因为 API 收费是按 Token 算的，你输入的提问算钱，它输出的回答也算钱。

#### 2. Prompt (提示词)
*   **我理解是**：就是你跟大模型说的==“一句话或一段话”==。吴恩达课上教的写清晰指令、给例子，都是在优化这个 Prompt。

#### 3. System Prompt / System Message (系统提示词)
*   **我理解是**：**给大模型设定的“人设”或“全局规则”**。
*   *举个例子*：在打电话前告诉客服“你接下来的身份是一个暴躁的四川大厨”。那么不管顾客问什么，他都会用这个身份回答。它决定了 AI 的==基调==、==语气==和==边界==。

#### 4. Messages (消息列表)
*   **我理解是**：**聊天记录的数组**。
*   在 API 开发中，我们不是直接发一句话给大模型，而是发一整个聊天记录的列表过去。这个列表里有三种角色：
    *   `system`: 人设（上面提到的）
    *   `user`: 用户的提问（比如“今天天气怎么样”）
    *   `assistant`: AI 历史的回答（比如“北京今天 25 度，晴天”）
*   **为什么重要**：这就是为什么大模型能“联系上下文”的原因，因为**你把之前的聊天记录又原封不动地发给了它**。

#### 5. Temperature (温度)
*   **我理解是**：控制大模型回答的“创造力”或“随机性”**的旋钮。取值范围通常是 0.0 到 2.0。
*   *举个例子*：
    *   **Temperature = 0**：每次问它“1+1=”，它永远只会回答“2”。（**适合写代码、做翻译、提取数据，要求确定性**）
    *   **Temperature = 0.8**：问它“给我讲个笑话”，它每次讲的笑话都不一样。（**适合写诗、头脑风暴、聊天，需要发散**）

#### 6. Top_p (核采样)
*   **我理解是**：它和 Temperature 的作用**一模一样**，都是控制==随机性==的。它是按概率门槛来筛选词汇的。
*   **工程建议**：**不要同时改这两个参数！** 业界的习惯是：**只改 Temperature，把 Top_p 永远固定为 1.0（或者 0.9，取决于模型默认值）**。你只要懂 Temperature 就够了。

#### 7. Context Window (上下文窗口)
*   **我理解是**：大模型的==“短期记忆容量”==**（金鱼脑）。
*   它能记住的上下文不是无限的，而是有最大 Token 数量限制的（比如 8k、32k、128k）。
*   **为什么重要**：如果你的聊天记录 + 你上传的文档 超过了这个窗口的容量，大模型就会“失忆”（报错，或者忘记最开头说的话）。

#### 8. Embedding (向量/嵌入)
*   **我理解是**：把人类语言翻译成**“高维空间的坐标系（一串很长的数字）”**。
*   *举个例子*：在数学空间里，“苹果”和“香蕉”的坐标离得很近（因为都是水果），“苹果”和“汽车”的坐标离得很远。
*   **为什么重要**：这是**未来第 5 周我们要学的 RAG（检索增强生成）的核心基础**！现在只要知道它是用来做==“语义相似度搜索”==的就行，不用深究。
