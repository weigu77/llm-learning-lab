### 📝 Day 5 每日复盘 (Daily Review)

**1. 今天我实际学会了什么？**
*   亲手跑通了 Temperature 的对比实验，直观感受到了大模型的“确定性”与“随机性”。
*   彻底弄懂了：当 Temperature 设为 0 时，模型每次只选概率最高的词，所以输出几乎一模一样；当 Temperature 调高时，模型开始选择低概率的词，从而产生创造力。
*   总结出了工程开发的黄金法则：**做提取、翻译、写代码等精准任务，必须用低 Temperature（0.0 - 0.2）；做头脑风暴、写诗等创意任务，才用高 Temperature（0.7+）。**

**2. 今天卡住的点是什么？**
*   整体很顺利。但我产生了一个新的思考：既然高 Temperature 会导致每次回答不一样，那在未来开发应用时，如果我希望 AI 的创意任务也能**保持稳定、不乱说**，除了调参数，我还应该怎么控制它？（这就需要用到 System Prompt 的强约束和第 2 周要学的结构化输出 JSON）。

**3. 明天最重要的一件事是什么？**
*   **周末重构日！** 不再写新的功能，而是要把前几天散乱的代码（比如 API 调用逻辑、配置管理）整理出来，封装成漂亮的、可以在其他项目里直接 import 复用的模块。像真正的软件工程师一样写代码！
